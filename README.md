📊 E-commerce Sales Data Analysis with SQL & Python

This project demonstrates analysis of an **e-commerce sales dataset** using **SQL queries** executed in **Jupyter Notebook**, connected to **MySQL via Python**. The dataset was large, so queries were run directly on the database for efficiency. Results are explained and visualized with Python libraries for clarity and actionable insights.

 🚀 Project Highlights
- Integration of **MySQL** with Jupyter Notebook using Python connectors.  
- Execution of **15 SQL queries** across basic, intermediate, and advanced levels.  
- Use of **visualizations** (bar charts, line graphs, pie charts) to present results.  
- Practical example of combining SQL’s querying power with Python’s analytical flexibility.  

 📂 Query Categories

🔹 Basic Queries
1. List all unique cities where customers are located.  
2. Count the number of orders placed in 2017.  
3. Find the total sales per category.  
4. Calculate the percentage of orders paid in installments.  
5. Count the number of customers from each state.  

🔸 Intermediate Queries
1. Calculate the number of orders per month in 2018.  
2. Find the average number of products per order, grouped by city.  
3. Calculate revenue contribution percentage by product category.  
4. Identify correlation between product price and purchase frequency.  
5. Rank sellers by total revenue generated.  

🔹 Advanced Queries
1. Moving average of order values per customer.  
2. Cumulative monthly sales for each year.  
3. Year-over-year growth rate of total sales.  
4. Customer retention rate (repeat purchase within 6 months).  
5. Top 3 customers by spending per year.  

📈 Purpose
This repository is designed for learners and professionals exploring **data analytics, SQL, and Python integration**. It highlights how to manage large datasets, extract insights, and present them in a reproducible format for real-world e-commerce analysis.
